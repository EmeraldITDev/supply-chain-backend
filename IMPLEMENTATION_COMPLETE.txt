🔧 QUICK START (5 MINUTES)

═══════════════════════════════════════════════════════════════════════════════

Step 1: Configure Email (choose one)

  Development (Testing):
  ────────────────────────────────────────────────────
  MAIL_MAILER=log
  FRONTEND_URL=http://localhost:3000

  Production (SendGrid - Recommended):
  ────────────────────────────────────────────────────
  MAIL_MAILER=smtp
  MAIL_HOST=smtp.sendgrid.net
  MAIL_PORT=587
  MAIL_USERNAME=apikey
  MAIL_PASSWORD=your_sendgrid_api_key
  MAIL_ENCRYPTION=tls
  MAIL_FROM_ADDRESS=noreply@yourdomain.com
  MAIL_FROM_NAME="${APP_NAME}"
  FRONTEND_URL=https://your-domain.com

Step 2: Clear Cache
  ────────────────────────────────────────────────────
  php artisan config:clear
  php artisan cache:clear

Step 3: Test (Optional)
  ────────────────────────────────────────────────────
  php artisan tinker
  
  use App\Services\EmailService;
  $emailService = app(EmailService::class);
  $emailService->sendVendorInvitation('test@example.com', 'Test Co');
  exit

Step 4: Use API
  ────────────────────────────────────────────────────
  curl -X POST http://localhost:8000/api/vendors/invite \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"email":"vendor@example.com","company_name":"ABC Ltd"}'

🔄 COMPLETE VENDOR ONBOARDING FLOW

═══════════════════════════════════════════════════════════════════════════════

1. Admin sends invitation
   └─> POST /api/vendors/invite
       ✉️ Email sent to vendor with registration link

2. Vendor registers
   └─> POST /api/vendors/register
       📋 Registration saved as "Pending"

3. Admin reviews & approves
   └─> POST /api/vendors/registrations/{id}/approve
       👤 Vendor account created
       ✉️ Email sent with login credentials

4. Vendor logs in (first time)
   └─> POST /api/vendors/auth/login
       🔒 Must change password

5. Vendor changes password
   └─> POST /api/vendors/auth/change-password
       ✅ Full access granted

6. Vendor updates profile
   └─> PUT /api/vendors/auth/profile
       ✅ Profile complete

📞 TROUBLESHOOTING

═══════════════════════════════════════════════════════════════════════════════

Issue: Emails not sending
  └─> Check .env configuration
  └─> Run: php artisan config:clear
  └─> Check: storage/logs/laravel.log
  └─> Verify email provider credentials

Issue: Emails going to spam
  └─> Verify sender domain
  └─> Add SPF/DKIM/DMARC records
  └─> Use professional from-address

Issue: 403 Forbidden on invite
  └─> Check user role (must be admin/procurement manager)
  └─> Verify Bearer token is valid

Issue: Can't update profile
  └─> Ensure using vendor token (not admin token)
  └─> Use PUT /api/vendors/auth/profile endpoint

See documentation for complete troubleshooting guide
